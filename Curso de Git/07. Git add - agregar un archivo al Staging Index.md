Cuando tienes un archivo nuevo que deseas que Git trackee o haces cambios sobre un archivo ya trackeado, estos cambios necesitan subirse al repositorio.

Para que un archivo sea commiteado, necesita estar en el Staging Index.

El comando `git add` se usa para mover archivos desde el Working Directory al Staging Index (más detalle de las secciones de git en [[01. Partes de Git]]). 

### Ejemplo de uso

Supongamos que tienes un archivo nuevo llamado `index.html`.

Para agregarlo al Staging Index, se ejecuta el siguiente comando:
```
git add index.html
```

![[ud123-l4-git-add-to-staging-recap.gif]]
_Aqui se aprecia el paso de archivos desde el directorio de trabajo al staging index._
### ¿Y siquiero agregar todos los cambios que he hecho?

Usa el siguiente comando:
```
git add .
```
