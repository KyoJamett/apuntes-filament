## 1. Working Directory: 
Capa donde se trabaja. Todos los archivos modificados se encuentran en el working directory.
## 2. Stage Index: 
Capa donde se almacenan los archivos que el programador sabe que no va a editar más. Estos son los archivos que irán en el commit. Para agregar un archivo al stage se usa el comando `git add [name]`, para subir todos los archivos modificados se usa el comando `git add .`.

## 3. Repository
Capa que contiene el registro de cambios del proyecto, aquí se almacenan los commits. El repositorio es local, y se debe comunicar con el repo remoto (alojado en github, gitlab, etc) para tener los cambios hechos por los demás devs del proyecto.
