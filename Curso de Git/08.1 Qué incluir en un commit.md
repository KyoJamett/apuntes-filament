## ¿Qué incluir?
El objetivo es que cada commit tenga un único objetivo. Cada commit debería almacenar un cambio unitario (el curso dice _single-unit change_). Si bien esto puede ser subjetivo, cada commit debería cambiar sólo un aspecto del proyecto.

Ahora bien, esto no limita el número de líneas de código que han sido agregadas o eliminadas, o el número de archivos agregados, editados o elimiandos. Digamos que quieres hacer cambios en una barra lateral para agregar una nueva imagen. Probablemente:
- Agregaste una nueva imagen al proyecto
- Modificaste el archivo de la vista
- Agregaste estilos al css

Un commit que incluya todas estas modificaciones sería perfectamente válido.

---
## ¿Qué no incluir?
Por el contrario, un commit no debería incluir cambios no relacionados entre si, por ejemplo cambios a la barra lateral y además un pequeño cambio de palabras en el footer. No se relacionan y no deberían ir en un mismo commit. 

Haz un cambio y su commit, luego el otro cambio y otro commit. Si uno de los commits sale mal, no perderás cambios que no están involucrados.

La mejor manera de saber qué debería ir en un commit es preguntarse "¿qué pasaría si todos los cambios del commit se borraran?". Si un commit se borra, se debería eliminar sólo una cosa.
