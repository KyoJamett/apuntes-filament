El comando `gti branch` se usa para interactuar con las ramas del repo.

```
git branch
```

Este comando se utiliza para:
- Listar todas las ramas del repositorio
- Crear nuevas ramas
- Eliminar ramas

Si sólo ejecutas `git branch`, se listarán las ramas del repo
![[Pasted image 20260213161529.png]]
_Al ejecutar git branch, se muestra la única rama creada, la rama master_

### Crear una rama
Para crear una rama usa el comando `git branch` y provee un nombre de la rama nueva.

Si quieres crear la rama desarrollo:
```
git branch desarrollo
```

Esto sin embargo, no nos sitúa en la rama desarrollo, sólo crea la rama nueva.

## Git checkout
El comando `git checkout` permite desplazarse a una rama del repo.

```
git checkout desarrollo
```

Este comando hace lo siguiente:
- Remueve todos los archivos del Working Directory que git está trackeando (que está almacenado en el repo).
- Va al repositorio y trae los archivos y carpetas del commit al que la rama está apuntando.

## Borrar una rama
Las ramas suelen usarse para hacer fixes de forma aislada del proyecto principal. Una vez has hecho los cambios, puedes combinar la rama a la rama `master` (spoiler: esto es un `merge`).

Una vez se han combinado, la rama del fix ya no es necesaria. Si quieres eliminar una rama, puedes hacerlo con el flag -d

```
git branch -d desarrollo
```
_Esto elimina la rama de desarrollo_

Toma en cuenta que no se  puede eliminar una rama sobre la cual estés situado. Para eliminarla debes situarte en una rama distinta (por ejemplo, `main`). Git no dejará que elimines una rama si hay commits en dicha rama, que no se encuentren en otra rama, esto previene que se pierdan commits que no se han mezclado.

Si quieres forzar el borrado igualmente, usa el siguiente comando:

```
git branch -D desarrollo
```
