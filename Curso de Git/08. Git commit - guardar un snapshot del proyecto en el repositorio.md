El comando `git commit` toma los archivos desde el staging index y los guarda en el repositorio.

```
git commit -m "mensaje"
```

Este comando adicionalmente va con el flag -m y el mensaje del commit entre comillas. Esto es para saltar el paso de que se abra el editor para escribir el mensaje. (En realidad, nunca aprendí a guardar un mensaje con el editor de texto, así que siempre uso este flag, pero diría que es práctico, además hoy en día los commits se administran mediante vs code).

---
### Hacer el commit sin usar el flag
- Se abrirá un editor de texto
- se debe escribir un mensaje en el archivo abierto
- Las líneas que inician con `#` son comentarios
- Guarda el archivo y cierra el editor
- El nuevo commit debería verse junto a los antiguos commits con el comando `git log`
