¿Cómo saber si deberías o no, correr un comando? ¿Tengo todo listo para hacer un commit? 
```
git status
```
_IMPORTANTE: úsalo siempre que prendas el pc y comiences a trabajar_

Este comando nos dirá lo que Git está pensando y el estado del repositorio. El output de este comando te mostrará si hay cambios en el working directory, o si hay archivos agregados al stage index ([[01. Partes de Git]]) 

### Ejemplo de output
```
On branch master 
Your branch is up-to-date with 'origin/master'. 
nothing to commit, working directory clean
```

Este output nos muestra lo siguiente
- `On branch master` : Nos indica que git se encuentra posicionado en la rama `master`.
- `Your branch is up-to-date with 'origin/master'.` : Nos indica que este proyecto se encuentra sincronizado con el proyecto desde donde lo copiamos (lo que no significa que esté al día, para eso necesitas hacer `git pull`).
- `nothing to commit, working directory clean` : Nos indica que no hay cambios en ningún archivo, no has modificado nada.
